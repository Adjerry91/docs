# VRCFaceTracking Parameters

The following tracking parameters are used by VRCFaceTracking to send face tracking 
data to avatars.

:::info VRCFaceTracking will detect prefixed parameter categories. 

Introduced with **Unified Expressions**, **VRCFaceTracking** has the ability to load parameters more dynamically.
Parameters can be loaded with prefixes, in the following ways:

`.../v2/JawOpen`  
`...ExamplePrefix/v2/JawOpen`  
`...Example/Nest/v2/JawOpen`  

This allows parameters to be organized much more effectively on avatars. Game engines such as 
**Unity** also automatically categorize parameter selections in the **Unity Editor**.

:::

:::note VRCFaceTracking uses Expression Parameters on Avatars to drive face tracking. 

VRCFaceTracking parameters can be used in the following forms: `Float` and `Binary`.

Learn more about the different [Parameter Types](/docs/category/types) here.

:::

:::caution Tracking parameters are explicitly defined.

Each tracking parameter references a Unified Expressions shape. 
For a reference as to what each shape does, please refer to the 
[Unified Expressions Base Shapes](../unified-blendshapes.mdx#ue-base-shapes).

:::

| v2/EyesX | `<-1.0 - 1.0>` Left to Right direction of eyes in normalized cartesian. | img |

| v2/EyeLeftX |  `<-1.0 - 1.0>` Left to Right of the left eye direction in normalized cartesian. | img |

| | |
